# 카드 메모리 게임 - 개발 문서

## 📚 문서 구성

이 폴더에는 카드 메모리 게임 프로젝트의 전체 개발 문서가 포함되어 있습니다.

---

## 📄 문서 목록

### 1. [요구사항 명세서](./01-requirements.md)
프로젝트의 전체 요구사항과 기능 명세

**주요 내용**:
- 프로젝트 개요 및 목표
- 핵심 기능 요구사항 (게임 모드, 카드 설정, 수익화)
- 사용자 경험 요구사항
- 기술적 요구사항
- 비기능 요구사항 (보안, 확장성, 테스트)

**대상**: 전체 팀원, 기획자, 개발자

---

### 2. [아키텍처 설계 문서](./02-architecture.md)
시스템 아키텍처 및 컴포넌트 구조

**주요 내용**:
- 전체 시스템 아키텍처
- 프론트엔드 디렉토리 구조
- 컴포넌트 계층 구조
- 상태 관리 아키텍처
- 게임 엔진 로직
- 성능 최적화 전략
- 보안 고려사항

**대상**: 시니어 개발자, 아키텍트, 테크 리드

---

### 3. [데이터 모델 및 API 명세서](./03-data-models.md)
TypeScript 타입 정의 및 API 설계

**주요 내용**:
- 게임 관련 타입 정의
- 덱(Deck) 관련 타입
- 사용자(User) 관련 타입
- 결제(Payment) 관련 타입
- 로컬 스토리지 스키마
- RESTful API 명세서 (향후 백엔드)
- 에러 코드 정의
- 데이터베이스 스키마 (참고용)

**대상**: 프론트엔드 개발자, 백엔드 개발자 (향후)

---

### 4. [개발 로드맵](./04-development-roadmap.md)
단계별 개발 계획 및 일정

**주요 내용**:
- **Phase 1**: Core MVP (4-6주) - 싱글 플레이 모드
- **Phase 2**: Enhanced Features (3-4주) - 대전 모드, 고급 기능
- **Phase 3**: Monetization (2-3주) - 포인트 시스템, 결제
- **Phase 4**: Polish & Launch (2-3주) - 최적화, 테스트, 배포
- 향후 계획 (백엔드, 소셜 기능, 모바일 앱)
- 리소스 요구사항
- 리스크 및 대응
- 성공 지표 (KPI)

**대상**: 프로젝트 매니저, 전체 팀원

---

### 5. [구현 가이드](./05-implementation-guide.md)
실제 코드 구현 예제 및 가이드

**주요 내용**:
- 개발 환경 설정
- 핵심 기능 구현 예제
  - 게임 상태 관리 (Context + useReducer)
  - 게임 엔진 로직
  - Custom Hooks
  - 컴포넌트 구현
- 스타일링 가이드 (Tailwind CSS)
- 테스트 작성 가이드
- 배포 가이드
- 성능 최적화 체크리스트

**대상**: 프론트엔드 개발자 (실무 담당자)

## 🛠️ 기술 스택

### 프론트엔드
- **React 19** - UI 라이브러리
- **TypeScript 5.8** - 타입 안전성
- **Vite 7** - 빌드 도구
- **Tailwind CSS 4** - 스타일링
- **Framer Motion** - 애니메이션
- **React Router** - 라우팅

### 상태 관리
- **Context API + useReducer** - 게임 상태 관리

### 테스트
- **Vitest** - 단위 테스트
- **Playwright** - E2E 테스트

### 배포
- **Vercel** 또는 **Netlify**
